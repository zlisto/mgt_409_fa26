Homework 1: Investor Due Diligence Extractor

Covers Lectures 1–2  |  Core MBA lens: Investor

Scenario

You are on the investment team at a growth equity fund. Associates spend hours pulling basic fields out of 10-K excerpts and earnings releases before a partner ever reads the memo. Your job: build a PDF → JSON extractor that a human investor can audit—not a chatbot that free-associates.

You will test on a public practice PDF (released on Canvas) and be graded on a hidden PDF (same format, different company). Variance comes from schema choices, handling missing data, numeric accuracy, and your written judgment—not from UI polish.

What you build

A CLI tool (required) that:

  1. Accepts a PDF path and output JSON path.
  2. Extracts investor-relevant fields into structured JSON matching your submitted schema.json.
  3. Never invents numbers: if a field is not supported by the document, use null and list the field in fields_not_found.
  4. Optionally exposes a simple web UI—but the autograder calls the CLI only.

Base JSON output contract

Every submission must produce an object with at least these keys (you may add more via schema.json):

{
  "company_name": "string",
  "fiscal_period": "string",
  "total_revenue_usd": number | null,
  "net_income_usd": number | null,
  "yoy_revenue_growth_pct": number | null,
  "risk_factors": ["string", "..."],
  "management_outlook_one_sentence": "string",
  "fields_not_found": ["string", "..."],
  "computed_metric": {
    "name": "string",
    "value": number | null,
    "formula": "string"
  }
}

Non-AI requirements (designed for variance)

  • Custom schema fields (≥2): You choose fields an investor would want (e.g., guidance_revenue_usd, segment_revenue_usd, debt_to_equity). Poor field choice = weak memo, not full autograder credit on hidden doc.
  • fields_not_found: Must list base fields you could not locate. Hidden PDF will omit at least one field—fabricating a value is a major penalty.
  • computed_metric: Pick one ratio or metric computable from extracted numbers (e.g., net margin = net_income / revenue). Show the formula string. If inputs are null, value must be null—do not divide by zero or guess.
  • risk_factors: At least 3 items; each must be a short phrase grounded in the filing (autograder checks overlap with source text on public/hidden PDFs).

Investor memo (memo.md)

Human-graded. Address all four prompts:

  1. Field you would not trust: One extracted field you would not rely on for a $10M investment decision—and why ( OCR, footnote, ambiguity, etc.).
  2. Custom fields rationale: Why you added your two schema fields; who on the deal team uses them.
  3. False precision: One example where the model looked confident but was wrong or unverifiable on the public PDF.
  4. Decision enabled: One concrete go / no-go / dig-deeper decision this tool speeds up on the public PDF.

Public practice PDF & self-test

Canvas will provide hw1_public.pdf. Before submitting, run:

python extract.py --pdf hw1_public.pdf --out sample_output/public_extract.json

The hidden grading PDF is not released. Do not hard-code company names or dollar amounts from the public file.

Autograder checks (instructor)

For transparency, automated tests include:

  • CLI exits 0 and produces parseable JSON
  • Output validates against student’s schema.json
  • All base fields present with correct JSON types
  • Numeric fields within ±2% of key figures in the test PDF (when present)
  • risk_factors length ≥ 3; each item has lexical overlap with PDF text
  • Fields documented as absent in instructor config → must be null and listed in fields_not_found
  • Penalty: any non-null value for a field marked absent in the hidden config
