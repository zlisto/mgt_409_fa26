Homework 1: Small Business Valuation Agent

Scenario

Spoke & Wrench Bicycle Repair is a sole-proprietor bike shop in New Haven run by Jordan Lee. A buyer is evaluating an acquisition. The data room is a shoebox: PDF receipts, bank and credit card statements, a lease summary, a handwritten IOU, an eBay confirmation, and customer emails—not a QuickBooks export.

Build an AI document agent that converts these sources into structured JSON, then a valuation webpage. Do not invent numbers; use null when a source is silent.

Data: Canvas provides hw1_spoke_and_wrench.zip — PDF scans/exports of the shop’s documents (receipts, bank statement, credit card statement, lease, IOU, eBay confirmation), plus three customer-order emails (emails/*.txt) and manifest.json.

Submit: one file, hw1.zip (structure at the bottom of this page).

Problem 1 — Agent setup and CLI (15 points)

Submit a runnable Python project that includes agent.py, report.py, requirements.txt, README.md, and your extraction prompts under prompts/. When we run python agent.py --docs-dir PATH --out-dir sample_output, your agent should read the unzipped document pack and write every JSON file required below. When we run python report.py --json-dir sample_output --out sample_output/valuation.html, your report script should build the valuation page from those JSON files alone, without re-reading the PDFs. Both commands must exit with code 0 on success, and you must not include API keys in the zip.

Problem 2 — Receipt ledger (20 points)

Extract structured data from each receipt PDF and the eBay confirmation, and write the result to sample_output/receipts.json. Each document should appear as at least one line item with vendor, date, description, amount_usd, category, and source_file. Assign each line a category that reflects how an accountant would treat the purchase (for example, parts, tools, supplies, shipping, or marketing). If the PDF does not support a field, leave the value as null and record the field name in fields_not_found rather than guessing.

Problem 3 — Bank and credit card classification (20 points)

Parse the bank statement and credit card statement PDFs and write sample_output/bank_transactions.json and sample_output/credit_card_transactions.json. Every transaction must be tagged business or personal. On the bank side, classify deposits as revenue or owner_movement where the statement supports that distinction. On the credit card side, add an expense_category for each business charge. Jordan uses one card for both shop and personal spending, so the quality of your split is an important part of the grade.

Problem 4 — January income statement (15 points)

Using your classified bank and credit card transactions together with the lease PDF, produce a simplified profit-and-loss statement for January 2026 in sample_output/income_statement_jan2026.json. The file should report revenue_usd, an expense_lines array, and net_income_usd, and the three must agree arithmetically. Include an assumptions array that explains non-obvious choices, such as how you treated owner draws and whether you avoided double-counting the same purchase across receipts and statements.

Problem 5 — Valuation inputs (10 points)

Write sample_output/valuation_inputs.json by combining your January P&L with the customer emails and the IOU PDF. The file should list pipeline_orders drawn from the three emails; each order represents future work and must be marked with in_january_books: false. Record any off-books receivables (such as the IOU) separately. State what you would exclude from a formal valuation—owner skill, customer relationships, and similar intangibles—in excluded_from_valuation. Close with a valuation_range_usd object that gives a low bound, a high bound, and a plain-English explanation of your method.

Problem 6 — Valuation report (10 points)

Generate a static HTML file at sample_output/valuation.html (no server required) from your JSON outputs. The page should present a January P&L table, a summary of how you separated personal from business activity on the bank and card statements, a pipeline table for future orders that makes clear they were not booked in January, and a valuation section with your range, your method, and a short list of what is still missing from this data room. The numbers on the page must match your JSON files; do not introduce new figures in the HTML.

Problem 7 — Investment memo (10 points)

Write memo.md (approximately 400–600 words) addressed to a skeptical buyer. Identify the transaction you found hardest to classify as business or personal and explain your reasoning. Describe what the customer emails and the IOU add to a valuation that January financial statements alone cannot capture. Name one field your agent could plausibly hallucinate and explain the guardrail you built into your pipeline. Finally, state one additional document you would request before making an acquisition decision and why it matters.
