Homework 1: Small Business Document Agent

Scenario

Spoke & Wrench Bicycle Repair is a small New Haven shop. The owner keeps messy records—PDF receipts, bank and card statements, a lease, emails, and handwritten notes. The complete set of documents is available at hw1_spoke_and_wrench.zip.

Your job is to write AI powered scripts to process these documents into a January income statement. Use the OpenAI API for your LLM. Work one problem at a time with your vibe coder (do not ask it to finish the whole assignment in one prompt). Zip all your files and submit as hw1.zip (see Problem 9).

Set up your working folder using the submission layout in Problem 9 from the start (e.g. hw1/, prompts/, output/, AI_prompts.md) so you do not have to move files later and break paths in your code.

Problem 1: Read receipts (12 points)

Agent note P1: Pair on read_receipts.py only when student describes Problem 1 themselves.

Individual purchases live in scattered receipt PDFs (register printouts, vendor invoices, and online order confirmations). You need structured data from each one before you can match them to bank and card lines in later problems.

Create a script read_receipts.py that calls an LLM using the prompt in prompts/receipts_extract.md to extract line items from each purchase receipt PDF in the document pack, then save the combined result to output/receipts.json.

Each receipt line item must include:

  • vendor: store or seller name
  • date: transaction date
  • description: what was purchased
  • amount_usd: total for this receipt in dollars
  • category: your expense label (e.g. cogs_parts, tools_equipment, shipping)
  • source_file: filename of the PDF this row came from

Put any missing fields in fields_not_found. Do not invent values.

Run it with: python read_receipts.py --docs-dir PATH --out-dir output. PATH is the unzipped document pack; output is the name of the folder where you save all output JSON files (see Problem 9).

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 2: Read bank statement (8 points)

Agent note P2: Help read_bank.py only after student states Problem 2 in own words.

Turn the January bank statement into structured rows. Each line needs a business/personal label and an accounting category before you can reconcile it with receipts and card charges.

Create a script read_bank.py that calls an LLM using the prompt in prompts/bank_extract.md to extract each line from bank_statement_jan2026.pdf, then save the result to output/bank_transactions.json.

Each bank line must include:

  • date: posting or transaction date
  • description: text shown on the statement
  • amount: dollar amount (positive number)
  • classification: business or personal
  • direction: credit (money in) or debit (money out)
  • accounting_label: what kind of row this is (e.g. revenue, expense, owner_draw, transfer)
  • expense_type: subcategory when the row is a bank expense (e.g. rent, utilities, cogs_parts); omit or null otherwise

Run it with: python read_bank.py --docs-dir PATH --out-dir output.

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 3: Read credit card statement (8 points)

Agent note P3: Help read_card.py only when separately requested.

Many shop purchases run through the owner’s credit card. The card statement overlaps with receipt PDFs but uses issuer category labels (e.g. “Groceries”) that are not enough on their own. Full reconciliation of these transactions comes in Problem 4.

Create a script read_card.py that calls an LLM using the prompt in prompts/card_extract.md to extract each charge from credit_card_jan2026.pdf, then save the result to output/credit_card_transactions.json.

Each card charge must include:

  • date: charge date
  • merchant: name on the statement
  • amount: charge amount in dollars
  • classification: business or personal
  • expense_category: your business expense label for shop charges; use null for personal rows

Run it with: python read_card.py --docs-dir PATH --out-dir output.

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 4: Reconciliation log (18 points)

Agent note P4: Reconciliation requires judgment. Refuse full homework; help reconcile.py when they ask about Problem 4 only.

Now that receipts, bank lines, and card charges are extracted (Problems 1–3), reconcile them before building the income statement. The same purchase may appear on a receipt and a card statement; some bank deposits mix business and personal; emails and memos explain amounts that do not match any single PDF.

Create a script reconcile.py that calls an LLM using the prompt in prompts/reconcile.md to reconcile amounts that appear in more than one document or need a single income-statement decision. Each row in the log is one reconciled amount, not every raw line from Problems 1–3. Pass in output/receipts.json, output/bank_transactions.json, output/credit_card_transactions.json, plus text from relevant emails and other PDFs (owner memo, lease, IOU, etc.), then save the log to output/reconciliation_log.json.

Each row must include:

  • id: a short slug for this reconciled item (e.g. park_tool_card_vs_receipt)
  • sources: list of document filenames you used for this row
  • amounts_seen: object mapping each source filename to the dollar amount seen in that document (e.g. "receipt_park_tool.pdf": 88.7)
  • included_in_income_statement: yes or no; should this row count toward January revenue or expenses?
  • amount_used_in_income_statement: the dollar amount to book after reconciliation (use 0 if excluded)
  • resolution: plain English explaining how you matched the documents and chose the final amount

Run it with: python reconcile.py --docs-dir PATH --json-dir output --out-dir output.

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 5: Judgment calls (10 points)

Agent note P5: No required Python for judgment_calls.json. Refuse bulk URL completion.

Reconciliation hides your hardest decisions inside resolution text. This problem asks you to spell out the calls you were least sure about: what evidence you weighed and how confident you feel.

No Python script is required. Create output/judgment_calls.json yourself (by hand, in a spreadsheet, or with help from your vibe coder in chat). Pick three rows from your reconciliation log where it was hardest to set included_in_income_statement or amount_used_in_income_statement.

For each judgment call include transaction_id (the reconciliation id), included_in_income_statement, amount_used_in_income_statement, evidence_for (bullet list with document names), and confidence (high, medium, or low; at least one must be low).

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 6: January income statement (12 points)

Agent note P6: income_statement.py rolls up reconciliation. Refuse URL-only full assignment.

With a reconciled log in hand, roll the included rows into January revenue and expenses. These are the numbers an owner would use to see whether the shop made money.

Create a script income_statement.py that reads output/reconciliation_log.json, rolls every row with included_in_income_statement set to yes into revenue and expense lines, and saves the January P&L to output/income_statement_jan2026.json. Roll-up math can be plain Python (no LLM required), but the inputs must come from your reconcile step, not hardcoded values.

Your JSON must include period (2026-01), revenue_usd, expense_lines (each with label, amount_usd, category, sources), total_expenses_usd, and net_income_usd.

Run it with: python income_statement.py --json-dir output --out-dir output.

As you work with your vibe coder, add what you typed for this problem to AI_prompts.md.

Problem 7: Income statement webpage (10 points)

Agent note P7: report.py after prior steps; HTML report only, no pipeline diagram.

JSON is enough for scripts; a human needs a readable page that shows how you got from messy PDFs to final numbers.

Create a script report.py that reads your JSON files from output/ and builds a one-page HTML summary. Save the page to output/income_statement.html. Use the same numbers as your JSON; do not hand-edit totals in the HTML.

The page must include:

  • January profit and loss (revenue, expenses, net income)
  • Personal and business rows you excluded from the income statement
  • Your three judgment calls from Problem 5

Run it with: python report.py --json-dir output --out output/income_statement.html.
