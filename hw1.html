Homework 1: Small Business Valuation Agent

Covers Lectures 1–2  |  Core MBA lenses: Accounting & Investor

Scenario

Spoke & Wrench Bicycle Repair is a sole-proprietor bike shop in New Haven run by Jordan Lee. A buyer is kicking the tires on an acquisition. The “data room” is a shoebox: PDF receipts, a bank statement, a credit card statement, a lease summary, a handwritten IOU, an eBay confirmation, and a few customer emails—not a clean QuickBooks export.

Your job: build an AI document agent that turns this mess into structured accounting records, then produces a valuation webpage. Strong work separates business from personal, knows what is not on the January books (future orders in email, off-books receivables, intangibles), and does not invent numbers.

This homework is inspired by Yale Accounting core pre-work (Prof. Rick Antle’s small-business valuation exercise).

Course document pack (Canvas)

Canvas provides hw1_spoke_and_wrench.zip with:

TypeFiles

Emails (.txt) 3 messages describing future customer orders (not January revenue)

What you build

  1. Document agent (CLI): Ingest PDFs and emails → structured JSON artifacts (see below).
  2. Valuation report (HTML): Summarize January economics, add pipeline and off-books items, and state a defensible valuation range for the shop.

Pipeline tasks (what the agent must do)

TaskInputOutput
Receipt ledger All receipt PDFs + eBay PDF List of line items with vendor, date, description, amount_usd, category
Bank split Bank statement PDF Transactions tagged business or personal; deposits classified as revenue vs owner movement
Card split Credit card PDF Charges tagged business or personal with expense category for business rows
January P&L (simplified) Your classified data + lease PDF Revenue, expense buckets, net income for January 2026 only
Valuation inputs Emails + IOU + your P&L Pipeline revenue, off-books receivables, and what you exclude (intangibles, etc.)

Receipt categories (minimum set)

Use these category values (add sub-notes in description if needed):

  • cogs_parts — inventory / parts for customer jobs
  • tools_equipment — durable shop tools (eBay stand, Park Tool)
  • shop_supplies — consumables, cleaning, bins
  • shipping
  • marketing
  • rent_occupancy — from lease / rent payment
  • utilities
  • insurance
  • software
  • uncategorized — only if the PDF truly does not support a label; explain in memo

Valuation webpage (valuation.html)

Single static HTML file (no server). Must include:

  1. January snapshot — simplified P&L table from your JSON.
  2. Personal vs business — summary of what you reclassified from bank/card.
  3. Pipeline table — future orders from emails (clearly labeled not booked in January).
  4. Valuation section — your range, method in plain English, and 2–3 drivers.
  5. What’s missing — bullet list of what a buyer would still need (intangibles, insurance detail, etc.).

CLI contract

python agent.py --docs-dir PATH_TO_UNZIPPED_PACK --out-dir sample_output
python report.py --json-dir sample_output --out sample_output/valuation.html

agent.py reads all PDFs in the pack plus emails/*.txt and writes the five JSON files. report.py builds the HTML from those JSON files only (no re-parsing PDFs).

Memo (memo.md, ~400–600 words)

  1. Hardest classification: One transaction you struggled to label business vs personal, and why.
  2. Not in the books: What the emails and IOU add to valuation that January financials alone miss.
  3. What you would not trust: One field your agent could hallucinate; how you guarded against it.
  4. Buyer diligence: If you were acquiring Spoke & Wrench, one follow-up document you would demand.
