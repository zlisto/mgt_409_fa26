Homework 1: Small Business Valuation Agent

Scenario

Spoke & Wrench Bicycle Repair is a sole-proprietor bike shop in New Haven run by Jordan Lee. A buyer is evaluating an acquisition. The data room is a shoebox: PDF receipts, bank and credit card statements, a lease summary, a handwritten IOU, an eBay confirmation, and customer emails—not a QuickBooks export.

Build an AI document agent that converts these sources into structured JSON, then a valuation webpage. Do not invent numbers; use null when a source is silent.

Data: Canvas provides hw1_spoke_and_wrench.zip (PDFs, emails/*.txt, manifest.json).

Submit: one file, hw1.zip (structure at the bottom of this page).

Problem 1 — Agent setup & CLI (15 pts)

Ship a runnable Python project with agent.py and report.py, requirements.txt, README.md, and prompts in prompts/.

  • python agent.py --docs-dir PATH --out-dir sample_output — reads the unzipped pack and writes all JSON outputs.
  • python report.py --json-dir sample_output --out sample_output/valuation.html — builds HTML from JSON only.
  • Both commands exit 0 on success. No API keys in the zip.

Problem 2 — Receipt ledger (20 pts)

Parse all receipt PDFs plus the eBay confirmation. Write sample_output/receipts.json.

  • One row per receipt/eBay document with vendor, date, description, amount_usd, category, source_file.
  • Choose a sensible category per line (e.g. parts, tools, supplies, shipping, marketing).
  • List missing fields in fields_not_found; do not guess amounts.

Problem 3 — Bank & credit card split (20 pts)

Parse the bank statement and credit card statement PDFs.

  • bank_transactions.json — every row tagged business or personal. Deposits labeled revenue vs owner_movement where applicable.
  • credit_card_transactions.json — each charge business or personal; business rows include expense_category.
  • Jordan mixes personal and business on one card—your split is graded.

Problem 4 — January income statement (15 pts)

From your classified bank/card data (and lease), write income_statement_jan2026.json for January 2026 only.

  • revenue_usd, expense_lines, net_income_usd must be internally consistent.
  • Include assumptions (e.g. how you treated owner draws, double-counting risk).

Problem 5 — Valuation inputs (15 pts)

Write valuation_inputs.json using emails, the IOU PDF, and your January P&L.

  • pipeline_orders from the three emails — future work, not January revenue (in_january_books: false).
  • off_books_receivables (e.g. IOU).
  • excluded_from_valuation — intangibles and gaps (owner skill, customer list, etc.).
  • valuation_range_usd with low, high, and a plain-English method.

Problem 6 — Valuation report & memo (15 pts)

sample_output/valuation.html (static HTML, no server) must include:

  1. January P&L table from your JSON.
  2. Summary of personal vs business reclassifications.
  3. Pipeline table (labeled not booked in January).
  4. Valuation range, method, and what is still missing from the data room.

memo.md (~400–600 words): (1) hardest business/personal call, (2) what emails/IOU add beyond January books, (3) one hallucination risk and your guardrail, (4) one follow-up document you would request as a buyer.
