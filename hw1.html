Homework 1: AI Earnings Screen

Covers Lectures 1–2  |  Core MBA lens: Investor

Scenario

You are on the investment team covering the AI semiconductor stack. Five companies just filed quarterly earnings press releases (SEC Form 8-K, Exhibit 99.1). Your associate pasted PDFs into a folder; your job is to turn unstructured earnings prose and tables into auditable JSON, then produce a comparison webpage a partner can read in five minutes—not a chat summary with invented numbers.

Canvas provides five PDFs: NVDA, AMD, AVGO, MU, QCOM. Fiscal quarters do not line up perfectly across filers; strong submissions say so explicitly instead of pretending the comparison is same-quarter.

What you build

Two deliverables:

  1. Extractor (CLI): One PDF in → one JSON file out with the required fields below.
  2. Comparison report (HTML): Reads your five JSON files and renders a single webpage that summarizes the data and gives an investment-style analysis (who looks strongest, what you would dig into next, what you do not trust).

Vibe coding is encouraged; you must be able to run and explain what you submit.

Required fields (per company JSON)

Each company file must be a JSON object with exactly these keys (use null when the press release does not state a value; never guess):

{
  "ticker": "NVDA",
  "company_name": "NVIDIA Corporation",
  "fiscal_period": "Q4 FY2026",
  "period_end_date": "2026-01-26",
  "revenue_usd": 39300000000,
  "revenue_yoy_pct": 78.0,
  "net_income_usd": null,
  "gaap_eps": 1.62,
  "non_gaap_eps": null,
  "ai_highlights": [
    "Short bullet grounded in the PDF",
    "Another bullet"
  ],
  "guidance_summary": "One sentence on forward guidance, or null",
  "fields_not_found": ["non_gaap_eps"],
  "computed_metric": {
    "name": "net_margin_pct",
    "value": null,
    "formula": "net_income_usd / revenue_usd * 100"
  }
}

Field rules

  • Dollars: Store full USD amounts as numbers (not strings). If the release states billions, convert to dollars (e.g., $39.3B → 39300000000).
  • Percentages: Store as numbers (e.g., 78.0 for 78% YoY revenue growth).
  • GAAP vs non-GAAP: Put GAAP EPS in gaap_eps; non-GAAP only in non_gaap_eps. Do not mix them.
  • ai_highlights: At least 2 short bullets that mention AI/data center/on-device AI themes and are traceable to the PDF wording.
  • fields_not_found: List every required key above (except computed_metric) that you could not locate. If inputs for computed_metric are missing, set value to null.
  • computed_metric: Use the net margin formula shown, or another metric computable only from your extracted numbers—document the formula string. No divide-by-zero; no fabricated inputs.

Comparison webpage requirements

Submit a single HTML file (no server required) that you generate from your JSON, e.g. sample_output/comparison.html. It must include:

  1. Summary table — all five companies, columns for ticker, fiscal period, revenue, revenue YoY %, GAAP EPS, and your computed metric.
  2. Rankings — you choose two metrics (e.g., revenue YoY % and net margin) and rank all five companies best → worst for each.
  3. Comparability caveat — a short paragraph on whether these quarters are apples-to-apples; if not, how you still use the screen.
  4. Analysis — 2–3 paragraphs: which name looks most attractive for an AI-infrastructure allocation, what risks the PDFs surface, and one field you would not trust without manual verification.

Styling is up to you; clarity matters more than design awards.

How to run (before you submit)

With the Canvas PDFs in a folder, e.g. pdfs/:

python extract.py --pdf pdfs/nvda-99.1.pdf --out sample_output/nvda.json
python extract.py --pdf pdfs/amd-99.1.pdf --out sample_output/amd.json
python extract.py --pdf pdfs/avgo-99.1.pdf --out sample_output/avgo.json
python extract.py --pdf pdfs/mu-99.1.pdf --out sample_output/mu.json
python extract.py --pdf pdfs/qcom-99.1.pdf --out sample_output/qcom.json
python report.py --json-dir sample_output --out sample_output/comparison.html

Open comparison.html in a browser to proofread numbers and narrative against the PDFs.

Memo (memo.md)

Address all three prompts:

  1. Hardest field: Which required field was hardest to extract reliably, and why (tables, GAAP/non-GAAP, units, etc.)?
  2. Comparability: How did fiscal-period mismatch affect your rankings or analysis?
  3. Tooling: One thing you would change in your prompt or pipeline before showing this to a portfolio manager.
